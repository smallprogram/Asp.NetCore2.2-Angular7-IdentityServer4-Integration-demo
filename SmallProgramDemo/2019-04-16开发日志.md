### asp.net core log API
Asp.net Core 提供了一套log API，内置6个log提供商<br>
1. Console
2. Debug
3. EventSource
4. EventLog
5. TraceSource
6. Azure App Service<br>
可以参考官方文档： https://docs.microsoft.com/zh-cn/aspnet/core/fundamentals/logging/?view=aspnetcore-2.2
### 第三方的Log提供商
1. Nlog
2. Serilog
3. ....
### 使用ILogger接口
通常使用依赖注入`ILogger<TCategoryName>`即可,TCategoryName为日志级别分类名

### ILogger的分类
1. 创建ILogger的时候指明分类
2. 分类名约定是调用类的全名(string)
3. 在ILoggerFactory上调用CreateLogger方法时可以指定分类的名称
4. 例如：
   > 方式一 `ILogger<TodoController> logger`<br>
   > 方式二 `logger.CreateLogger("TodoApi.Controllers.TodoController")`

### Asp.net Core 配置
#### 配置专用API(Configuration API)
1. Name-Value集合配置
2. 运行时可通过多个来源加载
3. 支持多层次结构

#### 配置的提供商
1. 文件格式（ini,Json,XML)
2. 命令行参数
3. 环境变量
4. 内冲中的.net队形
5. 未加密的Secret管理存储
6. 加密的用户存储，例如Azure私钥库
7. 自定义的提供商
#### 使用Json配置文件
1. 默认使用appsettings.json文件配置
2. 使用`ConfigurationBuilder().AddJsonFile("appsettings.json").Build() --> IConfigurationRoot<IConfiguration)`
#### 取得配置
1. 使用IConfiguration["Key:ChildKey"]
2. 针对"ConnectionString:xxx"，可以使用IConfiguration.GetConnectionString("xxx")来获取

#### 使用强类型配置(不常用)
参考微软官方文档

### 错误处理
使用app.UseDeveloperExceptionPage()自带的错误也方式进行错误处理也可以,针对MVC项目很好，但是针对API项目不是很友善。<br>
可以通过app.UseExceptionHandler()方法自定义自己的错误处理机制，返回状态码和相应的错误信息以及进行日志记录等操作。具体看代码<br>
可以自定义一个管道处理错误，自定义编写一个Extensions，将app.UseExceptionHandler()内的逻辑封装到该自定义管道中，之后在StartUp中调用即可。<br>
